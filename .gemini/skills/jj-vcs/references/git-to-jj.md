# Git と Jujutsu (jj) の比較

| アクション | Git コマンド | Jujutsu (jj) コマンド | 主要コンセプト |
| :--- | :--- | :--- | :--- |
| **状態確認** | `git status` | `jj st` | 作業コピー (`@`) とその親を表示します。 |
| **ログ表示** | `git log --graph` | `jj log` | Change ID 付きのコミットグラフを表示します。 |
| **ステージング** | `git add <file>` | (不要) | 無視されていないすべてを自動スナップショット。 |
| **メッセージ記録** | `git commit -m "..."` | `jj describe -m "..."` | 現在の作業コピーの説明を編集します。 |
| **確定・次へ** | `git commit` | `jj new` | 現在の上に新しい作業コピーを開始します。 |
| **変更を統合** | `git commit --amend` | `jj squash` | `@` の変更をその親に統合します。 |
| **破棄・戻す** | `git reset --hard` | `jj undo` | 最後の `jj` 操作を完全に取り消します。 |
| **切り替え** | `git checkout <branch>` | `jj new <bookmark>` | 特定のブックマークで新しい作業コピーを作成。 |
| **プッシュ** | `git push` | `jj git push` | 明示的にブックマークをリモートに送ります。 |
| **フェッチ** | `git fetch` | `jj git fetch` | リモートブックマークの更新を取得します。 |

## 重要な用語
- **作業コピー (Working Copy / `@`)**: あなたの作業スペースは、常に「進行中のコミット」です。
- **Change ID**: 「機能」に対する不変の ID（リベースや amend 後も変わりません）。
- **Commit ID**: 従来のハッシュ（内容やメッセージが変わると変化します）。
- **ブックマーク (Bookmark)**: Git のブランチに相当（ただしポインタは独立しています）。
- **Colocated (コロケート)**: `.git` と `.jj` が共存し、自動的に同期されるリポジトリ。
