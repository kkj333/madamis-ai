# Jujutsu (jj) VCS スキル

Git に代わるモダンなツール、Jujutsu (jj) を使用するためのガイドです。Jujutsu は作業ディレクトリ（Working Copy）をコミットとして追跡し、安全で強力な履歴操作を提供します。

## Git からの移行

既存の Git リポジトリで `.git` を同期させたまま `jj` を開始する方法（colocated モード）:

```bash
jj git init --colocate
```

これにより `.jj` ディレクトリが作成され、Git の HEAD が `jj` の作業コピー (`@`) と同期されます。

## 基本的なワークフロー

Jujutsu には「ステージング（git add）」のステップがありません。すべてのファイルの変更は自動的に現在の作業コピーのコミットに含まれます。

- **状態の確認**: `jj st`
- **ログ/グラフの表示**: `jj log`
- **説明（コミットメッセージ）の追加**: `jj describe -m "メッセージ"`
- **次のタスクを開始（現在の作業を確定）**: `jj new` (現在の上に新しい空のコミットを作成)
- **現在の変更を親に統合**: `jj squash`
- **最後の操作を取り消す**: `jj undo` (ミスを修正するのに非常に強力です)

## リモート操作

`jj` ではリモートブランチを「ブックマーク（bookmarks）」として管理します。

- **origin からフェッチ**: `jj git fetch`
- **ブックマークをプッシュ**: `jj git push --bookmark <名前>`
- **ブックマーク一覧を表示**: `jj bookmark list`

## エージェント（AI）向けのベストプラクティス

1. **変更の前後で必ず `jj st` を使用**し、作業コピーの状態を確認してください。
2. **`jj describe` を頻繁に使用**して、作業コピーの目的を明確に保ってください。
3. **`jj new` を推奨**して、新しい機能ごとに新鮮なコンテキストを開始してください。
4. **`jj squash` を使用**して、以前のタスクのコミットに修正を適用してください。
5. 複雑なリセットの代わりに **`jj undo` を活用**して、コマンドを間違えた場合に即座に復旧してください。
